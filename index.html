Gink

Description: Gink is a versioned, eventually consistent, multi-paradigm database management system. It takes a "protocol-first" approach, which facilitates multiple implementations that can share data. My personal contributions to Gink have included implementing new data structures, such as Role, PairMap, PairSet, KeySet, Box, and more, as well as creating a MemoryStore which caches the database in-memory using Tree Maps in TypeScript in order to improve read performance compared to IndexedDb. I have also handled most of the development operations, primarily integration testing in the browser, documentation generation and deployment, and automatic package publishing.

Technologies: JavaScript/TypeScript, Python, Docker, Linux (Debian/Ubuntu), Make, protocol buffers, Webpack, GitHub Actions (unit/integration testing, documentation generation, linting, and package publishing to PyPI and NPM).
